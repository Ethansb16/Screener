**Export 1: `resolveDocumentUrl(indexItems, baseUrl)`**
Pure function. Resolves which HTML document to fetch from the filing index directory items.

Priority order:
1. Find item where `item.type === 'EX-99.1'` — if multiple, pick highest `sequence` number (handles amendments per Pitfall 3)
2. Find item where `item.type === '10-12B'` or `item.type === '10-12B/A'`
3. Find first item where `item.name` ends with `.htm`
4. Return `null` if nothing found

Return value: full URL string (`${baseUrl}${item.name}`) or `null`.

Handle edge case: if `indexItems` is not an array, wrap in `[indexItems]` (Open Question 1 from research).

**Export 2: `fetchForm10Document(filing)`**
Async function. Accepts a filing object with `{ accession_number, primary_doc_url, cik }`.

Steps:
1. Compute `accessionNoDashes = accession_number.replace(/-/g, '')`
2. Build index URL: `${filing.primary_doc_url}${accessionNoDashes}-index.json`
3. Fetch index JSON via `edgarGetJson(indexUrl)` — wrap in try/catch, return `null` on failure with logger.warn
4. Extract items: `const items = indexData?.directory?.item ?? []`
5. Call `resolveDocumentUrl(items, filing.primary_doc_url)` to get the document URL
6. If no document URL, log warning and return `null`
7. Fetch the HTML via `edgarGetText(docUrl)`
8. Parse with `parse(html)` from node-html-parser
9. Return the parsed HTMLElement root

**CRITICAL rules:**
- NEVER call `fetch()` directly — use `edgarGetJson` and `edgarGetText` from edgarClient.js
- NEVER use `Promise.all()` — sequential fetches only
- `primary_doc_url` already ends with `/` (see edgarIngester.js line 81)

**Export: `locateSection(root, patternKey)`**
Implements the three-strategy section locator from research Pattern 2.

Parameters: `root` is a node-html-parser HTMLElement, `patternKey` is a key of SECTION_PATTERNS.

Strategy 1 — heading element search:
```javascript
const headings = root.querySelectorAll('h1,h2,h3,h4,h5,b,strong');
const heading = headings.find(el => pattern.test(el.text));
if (heading) return extractAfterHeading(heading);
```

Strategy 2 — bold paragraph fallback:
```javascript
const boldParas = root.querySelectorAll('p');
const match = boldParas.find(p => {
  const inner = p.querySelector('b,strong,font');
  return inner && pattern.test(inner.text);
});
if (match) return extractAfterHeading(match);
```

Strategy 3 — full-text search fallback:
```javascript
const fullText = root.text;
const idx = fullText.search(pattern);
if (idx !== -1) return fullText.slice(idx, idx + 5000);
return null;
```

**Helper (not exported): `extractAfterHeading(headingEl, maxChars = 5000)`**
Walk `nextElementSibling` collecting `.text` until hitting h1-h5 or exceeding maxChars. If collected text < 100 chars, also collect `.text` from sibling `<table>` elements (Pitfall 5: capitalization tables). Return trimmed text sliced to maxChars.

**Export: `classifyReasons(sectionText)`**
Pure function. Returns `{ classification, confidence }`.
- If `sectionText` is null/undefined/empty: return `{ classification: 'unknown', confidence: 'not_found' }`
- Count `strategicHits` = STRATEGIC_FOCUS_KEYWORDS.filter(r => r.test(sectionText)).length
- Count `weakHits` = WEAK_UNIT_KEYWORDS.filter(r => r.test(sectionText)).length
- If strategicHits >= 2 AND strategicHits > weakHits: `'strategic_focus'`
- Else if weakHits >= 1 AND weakHits > strategicHits: `'weak_unit_disposal'`
- Else if strategicHits >= 1 AND weakHits >= 1: `'mixed'`
- Else: `'unknown'`
- confidence: (strategicHits + weakHits) >= 3 ? 'high' : >= 1 ? 'medium' : 'low'

**Export: `classifyEquityGrants(sectionText)`**
Pure function. Returns `{ classification, confidence }`.
- Null/empty: `{ classification: 'unknown', confidence: 'not_found' }`
- Count matches in EQUITY_GRANT_KEYWORDS
- matches >= 2: `'equity_grants_confirmed'`, confidence based on count (>=3 high, else medium)
- matches === 0: `'no_equity_grants'`, confidence 'low'
- matches === 1: `'no_equity_grants'`, confidence 'low' (threshold is 2 per research to avoid false positives)

**Export: `classifyDebtLoading(sectionText)`**
Pure function. Returns `{ classification, confidence }`.
- Null/empty: `{ classification: 'unknown', confidence: 'not_found' }`
- Count matches in EXCESSIVE_DEBT_KEYWORDS
- matches >= 2: `'excessive_debt'`, confidence based on count (>=3 high, else medium)
- matches === 1: `'moderate_debt'`, confidence 'medium'
- matches === 0: `'no_debt_concern'`, confidence 'low'

**Export: `classifyManagement(sectionText)`**
Pure function. Returns `{ classification, confidence }`.
- Null/empty: `{ classification: 'unknown', confidence: 'not_found' }`
- Count `leadershipHits` = SPINCO_LEADERSHIP_KEYWORDS matches
- Count `retentionHits` = PARENT_RETENTION_KEYWORDS matches
- leadershipHits > retentionHits: `'strong_leaders_moving'`
- retentionHits > leadershipHits: `'leaders_staying_at_parent'`
- Both > 0 and equal: `'mixed'`
- Both 0: `'unknown'`
- confidence: (leadershipHits + retentionHits) >= 3 ? 'high' : >= 1 ? 'medium' : 'low'

**Export: `extractSignalsForFiling(filing)`**
Async function. Accepts a filing row object `{ id, accession_number, primary_doc_url, cik }`.

Steps:
1. Call `fetchForm10Document(filing)` to get parsed DOM root
2. If null, store all 4 signals with `classification: 'unknown', confidence: 'not_found'` and return
3. For each of the 4 signals:
   a. Call `locateSection(root, patternKey)` to get section text
   b. Call the appropriate classifier with the section text
   c. Call `storeSignal(filing.id, signalName, classification, confidence, sectionText)`
4. Return array of 4 signal result objects `[{ signal_name, classification, confidence }]`

Signal name mapping:
- `reasons_for_distribution` pattern key -> signal_name `'reason_classification'`, classifier `classifyReasons`
- `executive_compensation` pattern key -> signal_name `'equity_grants'`, classifier `classifyEquityGrants`
- `capitalization` pattern key -> signal_name `'debt_loading'`, classifier `classifyDebtLoading`
- `management` pattern key -> signal_name `'management_continuity'`, classifier `classifyManagement`
